# 하이퍼파라미터 스윕 결과 — 2026-04-29

> 스윕은 두 단계로 진행:
> - **1차** (`scripts/sweep.py`): 단일 train/val split 기준, 넓은 그리드 33 config
> - **2차** (`scripts/sweep_cv.py`): 5-fold CV 기준, 1차 베스트 영역의 finer grid 34 config
>
> 1차로 대략적인 영역을 식별 → 2차에서 CV 로 안정성 검증 + 미세 조정.

## 공통 셋업

- 데이터: `datasets/ev_battery_qc_train.csv` (13,565 행)
- 입력 피처 (6개, 표준화): `Ambient_Temp_C, Anode_Overhang_mm, Electrolyte_Volume_ml, Internal_Resistance_mOhm, Capacity_mAh, Retention_50Cycle_Pct`
- 타깃: `Defect_Type` (4 클래스, NaN → `"None"`)
- 클래스 분포: `None` 11,206 (82.6%) / `High Internal Resistance` 1,924 (14.2%) / `Poor Retention` 354 (2.6%) / `Critical Resistance` 81 (0.6%)
- 분할: stratified 80/20, seed=42 → train 10,852 / val 2,713
- 평가 지표: **macro-F1** 우선 (정확도는 imbalanced 데이터에서 majority 편향)

## 1차 스윕 — 넓은 그리드 + 단일 val

### 학습 셋업

- 학습 set: 위 train 에서 stratified 서브샘플 (1,848 rows — minority 클래스 cap)
- val set: 모든 config 동일 (2,713 rows)
- config 수: 33

### 1차 베스트 (변형별)

| 변형 | 베스트 config | val macro-F1 | val acc |
| --- | --- | --- | --- |
| 선형 hard-margin | `class_weight=none` | 0.7595 | 0.8496 |
| 선형 soft-margin | `C=100, balanced` | 0.7276 | 0.8290 |
| 비선형 RBF | `C=100, gamma=0.05, balanced` | 0.8247 | 0.9300 |
| 비선형 poly | `C=10, degree=4, balanced` | 0.8373 | 0.9337 |

### 1차 관찰

1. **하드마진이 소프트마진보다 약간 우위** — 데이터가 거의 선형 분리 가능에 가까움. 단 비선형 커널이 +0.08 macro-F1 더 끌어올림.
2. **gamma 민감도 (RBF)** — gamma=5.0 은 항상 오버피팅 (train=1.0 / val<0.45). 'scale' 또는 0.05 가 안정.
3. **degree=4 poly 가 변형 통틀어 1 위** — 6 차원 피처에 다항식 곡면이 잘 맞음.
4. **class_weight='balanced'** — soft/kernel 에서 효과 큼. hard-margin 에서는 slack 없어 미미.

→ 1 차 결과만으론 베스트가 단일 val 추첨에 의존했을 가능성 → **2 차 CV 스윕 으로 검증**.

## 2차 스윕 — 5-fold CV + finer grid

### 학습 셋업

- 5-fold stratified CV (각 fold 의 val 은 ~2,715 rows, train ~10,850)
- 각 fold 학습은 stratified 서브샘플 cap=4,000 (실효 ~3,800)
- 평가: **mean cv macro-F1 ± std** (5 fold 평균)
- config 수: 34
  - baselines 3 (linear-hard ×2 + soft C=100)
  - poly fine grid 15 (degree ∈ {3,4,5}, C ∈ {3,5,10,20,50})
  - rbf fine grid 16 (C ∈ {10,30,100,300}, gamma ∈ {0.02, 0.05, 0.1, 'scale'})

### 2차 관찰

1. **상위 10 개의 cv_f1 차이가 1σ 이내** — 베스트들은 통계적으로 거의 동급. 1 차의 "poly d=4 C=10" 도 여전히 강함 (랭크 2).
2. **CV 가 1 차 점수 전반을 끌어올림** — 단일 val 점수가 약간 운 나쁜 split 이었음. 진짜 일반화 성능은 CV 평균에 가까움.
3. **degree=3 가 충분** — d=4 와 통계적 동급, d=5 는 약간 떨어짐 (오버피팅 신호). 더 단순한 모델 선호.
4. **rbf C=300 이 acc 1 위 (0.9466)** — mF1 은 poly 1 위지만 acc 만 보면 rbf 가 약간 우위. 다른 트레이드오프.
5. **알고리즘 종류가 C/gamma/degree 보다 영향 큼** — 선형 (0.74-0.78) vs 비선형 (0.84) 차이가 0.05~0.10. 같은 알고리즘 안에서 hyper 조정은 ±0.005 수준.

## 최종 4 모델 선정 (CV 기반)

과제 요구 3 변형 + 비선형의 두 커널 비교를 위해 **4 모델** 학습:

| variant | config | 선정 사유 |
| --- | --- | --- |
| Linear hard-margin | `class_weight=none, max-train-samples=8000` | CV 베스트, cap 은 cvxopt 시간복잡도 회피 |
| Soft margin | `C=100, balanced` | CV 베스트 (1 차/2 차 모두 동일) |
| Kernel poly | `degree=3, C=50, gamma=scale, coef0=1.0, balanced` | **CV 전체 1 위** (1 차 베스트 d=4 C=10 에서 갱신) |
| Kernel rbf | `C=300, gamma=0.05, balanced` | CV rbf 1 위 (1 차 베스트 C=100 g=0.05 에서 갱신) |

각 변형을 **full train (10,852)** 에 다시 학습 → val 성능 비교는 [`04_results_2026-04-29.md`](04_results_2026-04-29.md) 참조.

## 부록 — sweep 메타-파라미터 영향 메모

`sweep_cv.py --train-cap 4000 --folds 5` 기준. 만약:
- `train-cap` 을 줄이면 (예: 2000): 학습 시간 ↓ 이지만 cv 점수 약간 낮아짐 (학습 데이터 부족)
- `folds` 를 늘리면 (예: 10): 추정 안정성 ↑ 이지만 시간 2 배. 5 가 일반적 sweet spot
- `seed` 를 바꾸면: top 5 안의 순서는 바뀌지만 알고리즘 종류 (poly vs rbf vs linear) 의 우열은 유지될 것

→ 본 결과는 **알고리즘 우열 (poly/rbf > linear)** 에 대해선 안정. **poly d=3 C=50 이 정확히 1 등인지** 는 어느 정도 운에 의존 (top 5 안의 차이가 σ 이내).

# 모델 비교 결과 — 2026-04-29

> 스윕(`03_sweep_2026-04-29.md`) 에서 선정한 4 변형을 full train (10,852 rows) 에 학습 후 동일 val set (n=2,713) 으로 비교.
>
> 다중 클래스 전략 두 가지 (OvR / OvO) 모두 시험. seed=42 고정. 학습 로그는 `runs/train_*.log` (gitignore).

## 셋업

- 분할: stratified 80/20, seed=42 → train 10,852 / val 2,713
- 클래스 분포 (val): None 2,241 / High IR 385 / Poor 71 / Critical 16
- 전처리: `StandardScaler` (모든 모델 공통), `LabelEncoder`
- 다중 클래스: One-vs-Rest (`MultiClassOvR`)
- 평가: val accuracy + val macro-F1 + per-class precision/recall + confusion matrix

## 학습 명령

```bash
# (1) Linear hard-margin
python train.py --variant linear --max-train-samples 8000 \
    --class-weight none --seed 42 --out models/linear_hard.pkl

# (2) Soft margin
python train.py --variant soft --C 100 \
    --class-weight balanced --seed 42 --out models/soft_C100.pkl

# (3) Kernel poly
python train.py --variant kernel --kernel poly --C 50 --degree 3 \
    --coef0 1.0 --gamma scale --class-weight balanced --seed 42 \
    --out models/kernel_poly_d3_C50.pkl

# (4) Kernel rbf
python train.py --variant kernel --kernel rbf --C 300 --gamma 0.05 \
    --class-weight balanced --seed 42 \
    --out models/kernel_rbf_C300_g005.pkl
```

> Linear hard 만 `max-train-samples=8000` (실효 3,887 — minority cap) — cvxopt QP 의 ill-conditioning(C=1e6) 회피.
> 나머지는 full 10,852 rows 학습.

| Linear | 2,395 | 49 (2.0%) | 2,346 (98.0%) | 위반자 거의 전부 — 선형 분리 한계 |

→ **margin SV 비율** 도 kernel 이 훨씬 높음 — soft margin 처럼 모든 점이 위반자가 아니라, 진짜 결정 경계 형성에 기여.

| 측면 | 효과 |
| --- | --- |
| Poor Retention recall 21% → 59% | + |
| Critical Resistance recall 94% → 50% | − |
| 전체 정확도 92.6% → 90.4% | − |

→ 선형 모델 + balanced 는 **Critical 손해 / Poor 이득** 의 단순 트레이드오프. 비선형으로 가야 둘 다 개선.

### 3. Poly vs RBF — 다른 실수 패턴

```

RBF          81.25%            83.10%        95.45%
```

- **None** 은 rbf 가 약간 더 보수적

→ **다른 실수를 함 = 앙상블 효과 가능성 높음**. 다음 단계에서 검증 가치 있음.

## OvR vs OvO 종합 비교

### val 정확도 / macro-F1

| 변형 | OvR (acc / mF1) | OvO (acc / mF1) | OvO 우위? |
| --- | --- | --- | --- |
| Linear hard-margin | 0.9255 / 0.770 | **0.9617 / 0.860** | ✅ 둘 다 |
| Soft margin C=100 | **0.9042** / 0.709 | 0.8699 / **0.794** | mF1 만 |
| Kernel poly d=3 C=50 | **0.9532** / 0.859 | 0.9506 / **0.863** | mF1 만 |
| Kernel RBF C=300 g=0.05 | **0.9565** / 0.847 | 0.9495 / **0.833** | acc 만 OvR |

### 핵심 발견

1. **Linear + OvO 가 의외의 1 위** — val_acc 0.9617. 비선형 커널보다 단순 선형 + OvO 전략이 더 정확.
2. **OvO 가 macro-F1 에서 일관 우위** — Critical / Poor 같은 소수 클래스 처리가 자연스럽게 균형 잡힘 (1대1 binary 안에서 두 클래스 비율이 큰 차이로 가지 않음).
3. **OvR 이 acc 에서는 살짝 우위** (RBF, poly) — 다수 클래스를 잘 맞추는 데는 OvR 도 충분.
4. **데이터의 본질적 결정 경계가 거의 선형** — 그래서 Linear + OvO 가 이김. 비선형 커널의 추가 표현력이 이 데이터에선 marginal.

## 부록 — 앙상블 탐색 (룰 1 위배 가능성으로 채택 안 함)

여러 모델의 결정함수를 합치는 앙상블 6 시나리오 시도:

| 시나리오 | 결합 | SVM 변형 측면 |
| --- | --- | --- |
| A1~A4 | 같은 변형의 OvR+OvO | 여전히 1 변형 → 룰 1 ✅ |
| B | linear_ovo + poly_ovo | 선형 + 비선형 혼합 → 룰 1 위배 ❌ |
| C, D, E, F | 다중 변형 결합 | 룰 1 위배 ❌ |

**판단**:
- A 시리즈도 단일 OvO 베스트 (linear_hard_ovo) 보다 향상 미미
- B-F 는 변형 혼합 → "선형 SVM, 선형 분리불가능 SVM, 비선형 SVM 구현" 룰의 spirit 위배 가능성

→ **앙상블 미채택, 변형별 단일 OvO 모델 제출**.

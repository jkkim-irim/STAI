# 최종 모델 비교 — 2026-04-29

> 스윕(`03_sweep_2026-04-29.md`)에서 선정한 변형별 베스트 config 를 train set 에 학습 후 동일 val set (n=2,713) 으로 비교한 결과.

## 셋업

- 분할: stratified 80/20, seed=42
  - train 10,852 (None 8,965 / High 1,539 / Poor 283 / Critical 65)
  - val   2,713  (None 2,241 / High 385  / Poor 71  / Critical 16)
- 모든 변형: 동일 `StandardScaler` + `LabelEncoder`
- 평가: 정확도(acc) + macro-F1(mF1) + per-class F1
- 모델 파일: `models/{linear_hard,soft_C100,kernel_poly_C10_d4}.pkl`

## 변형별 학습 명령

```bash
# Linear hard-margin (cap to 8000 stratified due to cvxopt cubic time)
python train.py --variant linear --max-train-samples 8000 \
    --class-weight none --out models/linear_hard.pkl

# Soft margin C=100, full train
python train.py --variant soft --C 100 \
    --class-weight balanced --out models/soft_C100.pkl

# Kernel polynomial degree 4, C=10, full train
python train.py --variant kernel --kernel poly --C 10 --degree 4 --coef0 1.0 \
    --gamma scale --class-weight balanced --out models/kernel_poly_C10_d4.pkl
```

> Linear hard 는 C=1e6 + 8,000 stratified subsample (실효 3,887 — minority cap). cvxopt 듀얼 QP 의 ~O(n²·³) 시간복잡도 + 대형 C 의 ill-conditioning 때문에 full 10,852 학습은 비현실적 (~1.5h). 작은 subsample 로도 가장 빠른 fit (111s) 에 best 단일 클래스 (Critical) 성능.

## 관찰 / 결론

1. **Kernel poly degree=4 가 모든 지표에서 베스트.** val acc 95.5%, val mF1 0.847, #SV 도 가장 적음 (1,716 → 가장 효율적). 6 차원 입력 공간에서 결정 경계가 4 차 다항식으로 잘 표현됨.

2. **Linear hard-margin** 은 단순함에 비해 매우 강함 (val acc 92.5%). 데이터가 거의 선형 분리 가능에 가까움을 시사. 다만 **Poor Retention recall 21%** 가 약점 — 50개를 None 으로 오분류 (`None ↔ Poor Retention` 의 결정 경계가 비선형).

3. **Soft margin (C=100, balanced weight)** 은 Poor Retention recall (59%) 은 끌어올렸지만 Critical recall (50%) 이 떨어짐 — 가장 작은 클래스가 균형 가중치에 오히려 손해 (소수 샘플로 큰 마진을 차지하려 하다 다른 소수 클래스와 충돌). #SV 가 7,957 로 가장 많아 (전체 train 의 73%) 모델 복잡도/추론 비용도 가장 높음.

4. **High Internal Resistance** 는 모든 모델에서 99%+ recall — 이 클래스는 입력 피처(특히 `Internal_Resistance_mOhm`) 와 직결되어 분류가 가장 쉬움.

5. **Poor Retention** 이 모든 모델의 약점 — Retention 컬럼이 None 클래스와 분포 겹침이 큼. kernel poly 가 0.507 F1 으로 가장 나은 처리.

## 제출용 모델 선정

**`models/kernel_poly_C10_d4.pkl`** — 최고 val acc, mF1, 모든 per-class F1, 가장 적은 #SV.

> 단, train mF1 = 0.913 vs val mF1 = 0.847 차이가 약간 크므로 (gap=0.066) 미공개 테스트셋에서는 약간 더 떨어질 가능성 있음. 보수적으로 보면 선형 hard-margin 의 단순함도 보험이 될 수 있음 — train/val gap = 0.012 로 거의 없음.
